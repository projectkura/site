## Basic delete

<Tabs items={["Lua", "TypeScript"]}>
<Tab value="Lua">
```lua
kura.db.delete('users'):where(op.eq('id', userId)):await()
```
</Tab>
<Tab value="TypeScript">
```ts
await db.delete(users).where(eq('id', userId));
```
</Tab>
</Tabs>

## Returning

Add a `RETURNING` clause to the generated SQL:

```lua
kura.db.delete('users')
  :where(op.eq('id', userId))
  :returning({ 'id', 'username' })
  :await()
```

<Callout type="warn">
  The current runtime still resolves delete queries to the affected row count. `returning()` is compiled into SQL, but the returned Lua/TypeScript value is not yet the returned row set.
</Callout>

## Soft deletes

A common pattern is to use a `deleted_at` column instead of actually removing rows:

```lua
kura.db.update('users')
  :set({ deleted_at = os.time() })
  :where(op.eq('id', userId))
  :await()

-- Query only active records
local active = kura.db.select()
  :from('users')
  :where(op.isNull('deleted_at'))
  :await()
```
