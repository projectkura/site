title: KuraDB
description: Asynchronous PostgreSQL wrapper with a typesafe Lua ORM for FiveM.
---

KuraDB is the database layer for Project Kura. It provides a high-performance, asynchronous PostgreSQL interface for FiveM with a chainable query builder, raw SQL helpers, schema-driven migrations, and auto-generated Lua types.

<Callout type="error" title="Compatibility">
  KuraDB is **not compatible** with `oxmysql`, `mysql-async`, or `ghmattimysql`. It is built exclusively for PostgreSQL. Remove any MySQL-based database resources before installing.
</Callout>

---

## Install PostgreSQL

1. Download from the [official website](https://www.postgresql.org/download/)
2. Install and note your **username**, **password**, and **port** (default `5432`)
3. Create a database for your server

```sql
CREATE DATABASE fivem;
```

## Download KuraDB

1. Get the latest release from [GitHub](https://github.com/projectkura/kuradb)
2. Extract into your server's `resources` directory
3. Ensure the folder is named `kuradb`

## Configure your server

Add to your `server.cfg`:

```bash
set kuradb_connection_string "postgres://root:password@localhost:5432/fivem"

ensure kuradb
```

<Callout type="warn" title="Special Characters">
  Avoid `;` `,` `/` `?` `:` `@` `&` `=` `+` `$` `#` in your database password — they have special meaning in connection URIs.
</Callout>

For Kura-native servers, use this start order:

```bash
ensure kura-lib
ensure kuradb
ensure kura-core
ensure kura-ui
```

## Add to your resource

<Tabs items={["Kura-native", "Standalone"]}>
<Tab value="Kura-native">
Add to your `fxmanifest.lua`:

```lua
shared_scripts {
    '@kura-core/init.lua',
}
```

`kura.db` and `op` are available automatically.
</Tab>
<Tab value="Standalone">
Add to your `fxmanifest.lua`:

```lua
server_scripts {
    '@kuradb/lib/KuraDB.lua',
    '@kuradb/lib/QueryBuilder.lua',
}
```
</Tab>

```json

  "Lua.runtime.version": "Lua 5.4",
  "Lua.diagnostics.globals": ["kura", "op"],

    "./kura-core",
    "./kura-lib",
    "./kura-ui",
    "./db/kuradb/lib"
  ]
}
```

## Your first query

Once your resource is set up, you can start querying immediately. `op` is available globally after loading `QueryBuilder.lua`.

```lua
kura.db.ready(function()
  local users = kura.db.select():from(kura.db.tables.users):await()
  print(#users .. ' users loaded')
end)
```

```lua
local users = kura.db.tables.users

-- Insert
kura.db.insert(users)
  :values({ username = 'john', email = 'john@example.com' })
  :await()

-- Select with filter
local cops = kura.db.select({ 'id', 'username' })
  :from(users)
  :where(op.eq('job', 'police'))
  :await()

-- Update
kura.db.update(users)
  :set({ username = 'jane' })
  :where(op.eq('id', userId))
  :await()

-- Delete
kura.db.delete(users)
  :where(op.eq('id', userId))
  :await()
```
